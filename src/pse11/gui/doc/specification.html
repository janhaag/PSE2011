Specification

A specification is one of the following:

  • assert
  • assume
  • ensure
  • invariant
  • axiom

Interpreting

When interpreting a progam, all specifications that are encountered during execution (with the exception of axioms) will be checked. If a specification fails, the program execution will stop; otherwise, the execution will continue as normal.

Validating

When validating a program, specifications (with the exception of axioms and assumes) are used to generate proof obligations. Axioms and assumes will be used to proof other specifications.
